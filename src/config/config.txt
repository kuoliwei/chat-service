========================================
Chat-Service 配置說明 (config.json)
========================================

{
  // ========================================
  // AI 相關設定
  // ========================================
  "ai": {

    // 各種操作的超時時間（單位: 毫秒）
    "timeouts": {
      // AI 生成回覆 / 摘要 / 存入摘要的最大等待時間
      // 超過時間 axios 會拋錯（被動報錯，中斷流程）
      // 位置: serviceClient.js - generateResponse() / generateSummary() / addSummary()
      "generateResponse": 120000,

      // RAG 初始化的最大等待時間（建立新聊天室時）
      // 位置: serviceClient.js - initializeRAG() / updateProtagonistRAG()
      "initializeRAG": 30000,

      // 清理 RAG 資料的最大等待時間（刪除聊天室 / 刪除摘要時）
      // 位置: serviceClient.js - cleanupRAG() / deleteSummaries()
      "cleanupRAG": 10000
    }
  },

  // ========================================
  // RAG 檢索設定（註：實際的檢索數量在 ai-service 的 config，
  //   此處參數目前 chat-service 未使用，保留備查）
  // ========================================
  "rag": {
    "topK": 3,
    "threshold": 0.7
  },

  // ========================================
  // 訊息摘要設定
  // ========================================
  "summary": {
    // 觸發摘要機制的字數閾值
    // 當「未摘要訊息（排除最新 shortTermLimit 條後）」的總字數 >= 200 時，觸發摘要
    // 流程:
    //   1. 撈出全部未摘要訊息
    //   2. 排除最新 shortTermLimit 條，其餘為候選
    //   3. 候選訊息總字數 >= threshold？是 → 摘要候選、標記為已摘要（記 summaryId）
    // 位置: conversationService.js - checkIfNeedsSummary()
    "threshold": 200,

    // 生成的摘要最多多少字（AI 生成摘要時被要求控制的上限）
    // 位置: conversationService.js - executeSummary()
    "maxWords": 20,

    // 摘要機制中不被摘要、保留原文的最新訊息數
    // 例: 未摘要有 8 條 → 只摘要前 6 條，保留最新 2 條原文
    // 這批「未摘要訊息」即短期記憶窗口，會完整送給模型（不再額外截斷）
    // 位置: conversationService.js - checkIfNeedsSummary()
    "shortTermLimit": 2
  }
}

========================================
典型場景示例
========================================

【場景 1】用戶發送新訊息
  1. 撈出全部未摘要訊息
  2. 檢查是否需要摘要（候選字數 >= threshold?），需要就先摘要
  3. 把「全部未摘要訊息 + 新訊息」送給 AI（短期記憶，不截斷）
     ＋ RAG 檢索的角色背景 / few-shot / 主角背景 / 歷史摘要
  4. AI 回覆，超時 120 秒（generateResponse）
  5. 成功才原子性存入用戶訊息 + AI 回覆

【場景 2】對話過長（需要摘要）
  未摘要訊息候選（排除最新 2 條後）總字數超過 200
  1. 摘要候選訊息 → 存入 Qdrant（得到 summaryId）
  2. 候選訊息標記 summarized=true、記下 summaryId
  3. 短期記憶只剩最新 2 條原文；老訊息改由 RAG 摘要檢索提供

【場景 3】RAG 檢索（實際發生在 ai-service）
  用戶說: "赫蘿，你昨天去哪？"
  1. 以當前訊息為索引，語義檢索角色背景 / few-shot / 歷史摘要
  2. 角色性格描述另以固定語意關鍵詞檢索（不隨當前對話漂移）
  3. 檢索結果放入 system prompt
  4. AI 用 system prompt + 未摘要對話生成回覆

========================================
修改建議
========================================

想提早觸發摘要（短期記憶保留更少、更快壓縮）？
  → 減小 summary.threshold（如 100）

想讓短期記憶保留更多原文？
  → 增大 summary.shortTermLimit（如 4）；或增大 threshold 延後摘要

想讓摘要更精簡 / 更詳細？
  → 調整 summary.maxWords

模型第一句慢 / 大模型生成久？
  → 增大 ai.timeouts.generateResponse（並同步前端輪詢上限）
